6 Constraints

6.1 The concept of constraints

Constraints and refiners are kind of two names for the same things. Well, depending on your definitions.

When solving a problem, a 'constraint' will be mapped into one or more low-level 'refiners'.

The choice of refiner(s) can vary depending on the input, and may be changed between versions of Vole as better refiners are created.

A constraint is a property that you can say whether or not any individual given permutation has that property.

6.3-1 VoleCon.InGroup
‣ VoleCon.InGroup( G )( function )

This constraint is satisfied by precisely those permutations in the group G.

gap> con1 := VoleCon.InGroup(DihedralGroup(IsPermGroup, 8));;
gap> con2 := VoleCon.InGroup(AlternatingGroup(4));;
gap> VoleFind.Group(con1, con2) = Group([(1,3)(2,4), (1,4)(2,3)]);
true

6.3-2 VoleCon.InCoset
‣ VoleCon.InCoset( U )( function )

This constraint is satisfied by precisely those permutations in the GAP right coset object U.

See also VoleCon.InLeftCoset (6.3-4) and VoleCon.InRightCoset (6.3-3), which allow a coset to be specifed by a subgroup and a representative element.

gap> true;
true

6.3-5 VoleCon.Stabilise
‣ VoleCon.Stabilise( object[, action] )( function )
‣ VoleCon.Stabilize( object[, action] )( function )

This constraint is satisfied by precisely those permutations that fix object under action, i.e. all permutations g such that action(object, g) = object. Note that such a stabiliser may be infinite.

The combinations of objects and actions that are supported by VoleCon.Stabilise are given in the following table.

The default action, in the case that the argument is not given, is OnPoints (Reference: OnPoints). This is the name in GAP of the action given by the ^ operator, i.e. it corresponds to object^g, where g in G. See \^ (Reference: ^).

Permitted action Corresponding object
OnPoints (Ref 41.2-1) [default] A point, or permutation, or perm group
OnTuples (Ref 41.2-5) A list of positive integers
OnSets (Ref 41.2-4) A set of positive integers
OnTuplesSets (Ref 41.2-10) A list of sets of positive integers
OnSetsSets (Ref 41.2-7) A set of sets of positive integers
OnDigraphs (Digraphs 7.1-1) A digraph object, or a list of adjacencies

6.3-6 VoleCon.Transport
‣ VoleCon.Transport( object1, object2[, action] )( function )

This constraint is satisfied by precisely those permutations that map object1 to object2 under action, i.e. all permutations g such that action(object1, g) = object2. Note that the set of such permutations may be infinite.

The combinations of objects and actions that are supported by VoleCon.Transport are given in the following table.

The default action, in the case that the argument is not given, is OnPoints (Reference: OnPoints). This is the name in GAP of the action given by the ^ operator, i.e. it corresponds to object^g, where g in G. See \^ (Reference: ^).

6.3-7 VoleCon.Normalise
‣ VoleCon.Normalise( G )( function )
‣ VoleCon.Normalize( G )( function )

This constraint is satisfied by precisely those permutations that normalise the permutation group G, i.e. that preserve G under conjugation.

Note that the set of such permutations is infinite.

gap> true;
true

6.3-8 VoleCon.Centralise
‣ VoleCon.Centralise( G )( function )
‣ VoleCon.Centralize( G )( function )

This constraint is satisfied by precisely those permutations that commute with G, if G is a permutation, or that commute with every element of G, if G is a permutation group.

Note that the set of such permutations is infinite.

gap> true;
true

6.3-9 VoleCon.Conjugate
‣ VoleCon.Conjugate( x, y )( function )

This constraint is satisfied by precisely those permutations that conjugate x to y, where x and y are either both permutations, or both permutation groups.

Note that the set of such permutations may be infinite.

gap> true;
true

6.3-10 VoleCon.MovedPoints
‣ VoleCon.MovedPoints( pointlist )( function )

This constraint is a shorthand for VoleCon.InGroup(SymmetricGroup(pointlist)). See VoleCon.InGroup (6.3-1).

gap> con1 := VoleCon.MovedPoints([1..5]);;
gap> con2 := VoleCon.MovedPoints([2,6,4,5]);;
gap> VoleFind.Group(con1, con2) = SymmetricGroup([2,4,5]);
true

6.3-11 VoleCon.LargestMovedPoint
‣ VoleCon.LargestMovedPoint( point )( function )

This constraint is a shorthand for VoleCon.InGroup(SymmetricGroup(point)). See VoleCon.InGroup (6.3-1).

gap> con := VoleCon.LargestMovedPoint(5);;
gap> VoleFind.Group(con) = SymmetricGroup(5);
true
